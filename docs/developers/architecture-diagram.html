9. Key Data Structures data_structures.h + repositories

TargetableDevice REPO

  • nodeId — uint32 unique ID (from packet)
  • configIndex — best freq/SF/BW combo
  • bestRSSI, avgRSSI, rssiStdDev — signal metrics
  • packetCount — total packets seen
  • originatedPackets / relayedPackets — traffic direction
  • protocol[16] — "Meshtastic", "LoRaWAN", etc.
  • deviceType[24] — hardware identification
  • powerClass — 0=low, 1=med, 2=high
  • periodicityScore — 0-100 beacon confidence
  • isRouter — relaying detected
  • Max 50 devices, LRU eviction of oldest inactive

CapturedPacket REPLAY

  • data[256] — raw packet bytes
  • length, configIndex — for TX config
  • originalRSSI, snr — capture metrics
  • nodeId, destId, packetId — parsed fields
  • hopCount, channel, priority — Meshtastic fields
  • decryptedText[256] — if PSK succeeded
  • Circular buffer of 10 slots

ScanConfig CORE

  • frequency — MHz (e.g., 906.875)
  • bandwidth — kHz (125, 250, 500)
  • spreadingFactor — 7-12
  • codingRate — 5-8
  • syncWord — protocol sync
  • 29 configurations swept in RECON mode

PacketInfo CORE

  • protocol — identified type
  • nodeId, destId — source/dest
  • rssi, snr — RF metrics
  • hopCount, channel — mesh fields
  • isEncrypted, decryptedText — PSK result
  • deviceType, firmwareVersion — ID
10. Concurrency & Safety Model

Dual-Core ESP32-S3

  • Core 0: Main loop — radio RX, packet processing, display, GPS
  • Core 1: AsyncWebServer — HTTP handlers, WebSocket
  • ISR context: DIO1 interrupt — sets atomic flag only

Protection Mechanisms

  • std::atomic<T> for totalPackets, droppedPackets, mode
  • ScopedLock (FreeRTOS mutex, 100ms timeout) for DeviceRepository
  • StaticRingBuffer — lock-free single-producer queue
  • Watchdog timer (30s) auto-reboots on hang
11. Persistence Layers

NVS (Non-Volatile Storage)

  • Operation mode + target config
  • Crash context (last action, heap, uptime)
  • WiFi credentials (encrypted)
  • API token
  • Device unique ID
  • Saved every 5 min (flash wear limit)

SD Card (T3-S3 / T-Beam)

  • PacketLogger — CSV per session
  • PCAP export files
  • GPS track logs
  • Session metadata

LittleFS

  • Web app files (HTML/CSS/JS)
  • WiFi credentials JSON
  • Served by ESPAsyncWebServer
  • Uploaded via pio uploadfs
